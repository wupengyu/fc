## 1. 项目概述

### 1.1 项目背景

本项目是基于千寻微信框架Pro的报单统计增强模块，旨在为彩票代购站点提供微信群报单的实时统计能力。

**业务场景**：
- 用户在微信群中发送彩票投注信息（报单）
- 系统缓冲15秒后批量处理报单消息
- 批量调用AI解析报单内容
- 前端定时轮询（5秒）获取统计数据刷新页面

**目标用户**：
- 彩票代购站点管理员
- 报单群运营人员

### 1.2 项目目标

1. **自动识别**：从微信群消息中自动识别彩票报单
2. **批量解析**：缓冲15秒后批量调用AI解析报单
3. **实时统计**：写时更新号码热度、金额分布统计
4. **前端轮询**：前端5秒定时拉取最新统计数据

### 1.3 核心功能

| 功能模块 | 描述 | 优先级 |
|---------|------|-------|
| 消息缓冲 | ConcurrentHashMap缓冲15秒（可配置） | P0 |
| 报单识别 | 关键词+模式匹配预筛选 | P0 |
| 批量AI解析 | 批量调用OpenAI兼容接口解析报单 | P0 |
| 统计汇总 | 写时更新的号码统计（UPSERT） | P0 |
| 统计查询 | REST API供前端轮询 | P0 |
| 外部接入 | REST API支持补录/回放报单 | P2 |

### 1.4 设计原则

1. **简单优先**：单机缓存，不考虑宕机数据恢复
2. **批量处理**：缓冲后批量AI解析，减少API调用次数
3. **前端轮询**：不使用WebSocket，前端5秒定时拉取
4. **丢失容忍**：缓存消息丢失可接受，不做复杂恢复机制

### 1.5 术语定义

| 术语 | 定义 |
|-----|------|
| 报单 | 用户在微信群发送的彩票投注信息文本 |
| FC | 福彩（福利彩票），如双色球、3D、七乐彩 |
| TC | 体彩（体育彩票），如大乐透、排列三、排列五 |
| 3D | 福彩3D，三位数字彩票 |
| P3/P5 | 体彩排列三/排列五 |
| SSQ | 双色球（Shuang Se Qiu） |
| DLT | 大乐透（Da Le Tou） |
| issue_key | 统计期维度：期号（如2026001）或日期（如2026-01-30） |
| number_zone | 号码区：MAIN/RED/BLUE/FRONT/BACK/POSITION |
| fingerprint | 幂等指纹，用于报单去重的SHA256哈希值 |

## 2. 总体架构设计

### 2.1 现有架构（保持不变的链路）

当前项目基于千寻微信框架Pro的 Spring Boot 应用（Java 21 + Spring Boot 3.3.5），核心链路：

```
微信群消息 → 框架回调 → CallBackController(/callback)
    → event分发(SpringBean: "event:10008")
    → RecvMsgGroupEvent
    → MessageBufferService(ConcurrentHashMap缓冲)
    → MessageFlushTask(15秒落库) → t_message
```

### 2.2 扩展架构（新增报单统计链路 - 批量处理）

新增报单处理链路，**复用现有的缓冲机制**，批量处理：

```
微信群消息 → RecvMsgGroupEvent
    └── OrderBufferService.add(msg)  // 放入报单缓冲区(ConcurrentHashMap)

OrderFlushTask @Scheduled(15秒，可配置)
    │
    ├─ Step1: atomicSwap() 原子交换取出缓冲区
    │
    ├─ Step2: OrderDetector.filter() 筛选疑似报单
    │
    ├─ Step3: 批量入库 → t_order_raw（幂等fingerprint）
    │
    ├─ Step4: 批量调用AI解析（一次请求，多条报单）
    │
    ├─ Step5: 拆分落库
    │         → t_order_parse_batch
    │         → t_order_item
    │         → t_order_item_number
    │
    └─ Step6: UPSERT → t_number_stats（统计汇总）

前端轮询：setInterval(5秒) → GET /api/stats → 刷新页面
```

### 2.3 新增入口

* `POST /api/orders/raw`：外部报单接入（可用于补录、回放、联调）
* `GET  /api/stats`：统计查询（TopN、排序、维度过滤）- **前端轮询此接口**

## 4. 数据模型设计

### 4.1 设计原则

1. **原始数据不可丢**：保留 raw 原文，便于审计与重放
2. **解析批次可追溯**：AI每次解析形成 batch，记录版本/耗时
3. **投注项与号码拆分**：报单→投注项→号码粒度三层拆分
4. **汇总写时更新**：实时统计通过 UPSERT 写入 `t_number_stats`
5. **简单优先**：不做复杂的分布式锁和重试机制

## 6. 核心流程（批量处理）

### 6.1 消息缓冲（OrderBufferService）

* 使用 `ConcurrentHashMap<String, OrderMessage>` 缓冲消息
* Key为 `msgId` 或 `fingerprint`，天然去重
* 宕机丢失缓存：**可接受，不做持久化**

```java
@Service
public class OrderBufferService {
    private final ConcurrentHashMap<String, OrderMessage> buffer = new ConcurrentHashMap<>();

    public void add(OrderMessage msg) {
        buffer.put(msg.getFingerprint(), msg);
    }

    public List<OrderMessage> swapOut() {
        // 原子交换：取出所有并清空
        List<OrderMessage> list = new ArrayList<>(buffer.values());
        buffer.clear();
        return list;
    }
}
```

---

### 6.2 定时批量处理（OrderFlushTask）

**核心流程**：缓冲 → 筛选 → 入库 → AI解析 → 拆分落库 → 统计

```java
@Scheduled(fixedDelayString = "${order.flush-interval:15000}")
public void flush() {
    // Step1: 原子交换取出缓冲区
    List<OrderMessage> messages = orderBufferService.swapOut();
    if (messages.isEmpty()) return;

    // Step2: 筛选疑似报单
    List<OrderMessage> orders = orderDetector.filter(messages);
    if (orders.isEmpty()) return;

    // Step3: 批量入库 t_order_raw（幂等）
    List<OrderRaw> rawList = orderIngressService.batchIngest(orders);

    // Step4: 批量调用AI解析
    List<AiParseResult> results = aiParseService.batchParse(rawList);

    // Step5: 拆分落库 + 统计
    normalizePersistService.batchPersistAndStat(rawList, results);
}
```

### 6.3 报单识别（OrderDetector）

* 目标：**高召回**，宁可误判交给AI失败，也不要漏报单
* 规则：
  1. 命中关键词（可配置）：双色球、大乐透、3D、排列三...
  2. 命中单位词 + 数字：单/组/米/倍 + 数字
  3. 典型 3D/P3：三位数字 + 单/组/金额
  4. 双色球/大乐透宽松结构

```java
public List<OrderMessage> filter(List<OrderMessage> messages) {
    return messages.stream()
        .filter(this::isLikelyOrder)
        .collect(Collectors.toList());
}

private boolean isLikelyOrder(OrderMessage msg) {
    String text = msg.getRawText();
    // 关键词匹配
    if (containsKeyword(text)) return true;
    // 模式匹配：数字+单位词
    if (PATTERN_ORDER.matcher(text).find()) return true;
    return false;
}
```

### 6.4 批量AI解析（AiParseService）

**批量请求**：一次API调用解析多条报单，减少调用次数

```java
public List<AiParseResult> batchParse(List<OrderRaw> rawList) {
    // 构造批量请求
    String batchPrompt = buildBatchPrompt(rawList);

    // 调用AI
    String response = callAiApi(batchPrompt);

    // 解析响应，拆分为每条报单的结果
    return parseBatchResponse(response, rawList);
}
```

**AI Prompt 示例（批量）**：

```
你是彩票报单解析专家。请解析以下多条报单，返回JSON数组。

报单列表：
[1] 512四单一组10
[2] 123一单4米
[3] 双色球 01 02 03 04 05 06 + 07 2注

返回格式：
[
  {"index": 1, "lottery_category": "FC", "game_type": "3D", ...},
  {"index": 2, ...},
  ...
]
```

### 6.5 拆分落库（NormalizePersistService）

```java
@Transactional
public void batchPersistAndStat(List<OrderRaw> rawList, List<AiParseResult> results) {
    for (int i = 0; i < rawList.size(); i++) {
        OrderRaw raw = rawList.get(i);
        AiParseResult result = results.get(i);

        if (result.isFailed()) {
            log.warn("AI解析失败: rawId={}, error={}", raw.getId(), result.getError());
            continue; // 跳过失败的，简单处理
        }

        // 1. 写入解析批次
        OrderParseBatch batch = saveParseBatch(raw, result);

        // 2. 拆分投注项
        for (ParsedOrder order : result.getOrders()) {
            OrderItem item = saveOrderItem(raw, batch, order);

            // 3. 拆分号码
            for (String number : order.getNumbers()) {
                saveOrderItemNumber(item, order.getNumberZone(), number);
            }

            // 4. 闸门检查 + 统计更新
            if (tryApplyStat(item)) {
                upsertNumberStats(item);
            }
        }
    }
}
```

## 10. 事务与一致性策略

1. **幂等入库**：`t_order_raw.fingerprint` 唯一键
2. **统计闸门**：`t_order_item.stat_applied` 保证同一item只入统计一次
3. **事务边界**：拆分落库 + stats UPSERT 在同一事务内
4. **失败处理**：AI解析失败仅记录日志，跳过该条，不阻塞整体流程

## 17. 前端轮询示例

```html
<script>
const API_URL = '/wechat/api/stats';
const POLL_INTERVAL = 5000; // 5秒

async function fetchStats() {
    const params = new URLSearchParams({
        lotteryCategory: 'FC',
        gameType: '3D',
        issueKey: '2026-01-30',
        sortBy: 'count',
        topN: 20
    });

    const response = await fetch(`${API_URL}?${params}`, {
        headers: { 'X-API-Key': 'your-api-key' }
    });
    const result = await response.json();

    if (result.success) {
        renderStats(result.data);
    }
}

function renderStats(data) {
    // 更新页面统计数据
    document.getElementById('stats-table').innerHTML =
        data.stats.map(s => `<tr><td>${s.rank}</td><td>${s.number}</td><td>${s.orderCount}</td></tr>`).join('');
}

// 启动轮询
setInterval(fetchStats, POLL_INTERVAL);
fetchStats(); // 立即执行一次
</script>
```
