# 福彩/体彩报单实时统计系统 — 开发设计文档

## 1. 整体架构设计

### 1.1 现有架构

当前项目是基于千寻微信框架Pro的Spring Boot应用（Java 21 + Spring Boot 3.3.5），核心链路：

```
微信群消息 → 千寻框架回调 → CallBackController(/callback)
    → 按event类型分发(SpringBean: "event:10008")
    → RecvMsgGroupEvent处理
    → MessageBufferService(ConcurrentHashMap缓冲)
    → MessageFlushTask(定时15秒刷写) → t_message表
```

### 1.2 扩展架构

在现有架构上新增报单统计链路，与原消息存储链路并行：

```
微信群消息 → RecvMsgGroupEvent
    ├── [现有] MessageBufferService → t_message (不变)
    └── [新增] OrderProcessService (报单识别+处理)
              ├── 1. 幂等检查 + 保存原始报单 → t_order_raw
              ├── 2. @Async → AiParseService (AI解析)
              │       ├── 成功 → t_order_normalized + UPSERT t_number_stats
              │       │         → StatsNotifyService → WebSocket推送
              │       └── 失败(重试3次) → t_ai_parse_fail_queue
              └── 3. 返回(不阻塞主链路)

新增入口：
  POST /api/orders/raw        → OrderController (外部报单接入)
  GET  /api/stats              → StatsController (统计查询)
  WS   /ws/stats               → StatsWebSocketHandler (实时推送)
  GET  /api/admin/parse-queue  → AdminController (失败队列管理)
```

### 1.3 技术选型补充

| 能力 | 选型 | 理由 |
|------|------|------|
| WebSocket | spring-boot-starter-websocket | Spring原生 |
| HTTP客户端(AI调用) | Hutool HttpUtil | 项目已有 |
| AI服务 | 第三方OpenAI兼容接口(cpass.cc) | 使用gpt-5.2模型 |
| 异步执行 | @Async + ThreadPoolTaskExecutor | 轻量 |
| 接口鉴权 | 自定义拦截器 + API Key | 不引入Spring Security |
| 前端 | 内嵌静态HTML + ECharts(CDN) + 原生WebSocket | 无需构建工具 |

新增pom依赖：`spring-boot-starter-websocket`

## 4. 核心流程

### 4.1 报单识别规则

在调用AI之前，先通过关键词快速判断消息是否可能是报单：

```java
public class OrderDetector {

    // 彩种关键词
    private static final Set<String> LOTTERY_KEYWORDS = Set.of(
        "双色球", "ssq", "红球", "蓝球",
        "大乐透", "dlt", "前区", "后区",
        "3d", "3D", "福彩3d", "福彩3D", "三d", "三D",
        "排列三", "排列3", "排三", "p3", "P3",
        "排列五", "排列5", "排五", "p5", "P5",
        "七乐彩", "7乐彩", "七星彩", "7星彩"
    );

    // 数字模式：至少包含3个以上数字
    private static final Pattern NUMBER_PATTERN =
        Pattern.compile("\\d{1,2}[\\s,，、+\\-|]+\\d{1,2}[\\s,，、+\\-|]+\\d{1,2}");

    public static boolean isLikelyOrder(String msg) {
        if (msg == null || msg.length() < 5) {
            return false;
        }
        String lowerMsg = msg.toLowerCase();

        // 规则1：包含彩种关键词
        for (String keyword : LOTTERY_KEYWORDS) {
            if (lowerMsg.contains(keyword.toLowerCase())) {
                return true;
            }
        }

        // 规则2：包含期号关键词 + 数字模式
        if ((lowerMsg.contains("期") || lowerMsg.contains("第"))
            && NUMBER_PATTERN.matcher(msg).find()) {
            return true;
        }

        // 规则3：纯数字模式（6+1或5+2结构，可能是双色球/大乐透）
        if (msg.matches(".*\\d{2}\\s+\\d{2}\\s+\\d{2}\\s+\\d{2}\\s+\\d{2}\\s+\\d{2}\\s*[+|]\\s*\\d{2}.*")) {
            return true; // 6+1 结构
        }
        if (msg.matches(".*\\d{2}\\s+\\d{2}\\s+\\d{2}\\s+\\d{2}\\s+\\d{2}\\s*[+|]\\s*\\d{2}\\s+\\d{2}.*")) {
            return true; // 5+2 结构
        }

        return false;
    }
}
```

### 4.2 微信群报单链路

```
RecvMsgGroupEvent.handle()
  ├── [现有] messageBufferService.addMessage(data)
  └── [新增] OrderDetector.isLikelyOrder(msg) 快速预筛选
        │ 否 → 跳过，不调用AI
        │ 是 ↓
        └── OrderProcessService.processFromWechat(data)
              ├── 幂等检查(msg_id) → 已存在则跳过
              ├── INSERT t_order_raw
              └── @Async → AiParseService.parseAndSave(rawId, rawText)
                    ├── 构造Prompt + 调用AI API (重试3次，指数退避1s/2s/4s)
                    ├── AI返回解析结果
                    ├── OrderValidator.validate() 校验号码格式
                    ├── 成功: INSERT t_order_normalized + UPSERT t_number_stats
                    │        → WebSocket广播
                    └── 失败: INSERT失败记录 + 入t_ai_parse_fail_queue
```

### 4.3 失败重试

```
AiParseRetryTask (每60秒)
  → 查询status=0且retry_count<10的记录(LIMIT 20)
  → 逐条重新调用AI解析
  → 成功→status=1, 失败→retry_count++, 超10次→status=2(放弃)
```

#### 6.3.1 System Prompt（完整版 - 针对3D/排三报单优化）

```text
你是一个专业的中国彩票报单解析助手，专门处理福彩3D和体彩排列三的报单。你的任务是将用户发送的非结构化报单文本解析为标准JSON格式。

## 一、彩种识别规则

1. **福彩3D (FC/3D)**：
   - 关键词："福彩"、"福"、"3D"、"福3D"
   - 无明确标注时默认为福彩3D

2. **体彩排列三 (TC/P3)**：
   - 关键词："体彩"、"体"、"排三"、"排列三"、"P3"、"体排三"

## 二、玩法类型识别

### 2.1 直选单式
- 格式：3位数字 + 单数/组数 + 金额
- 示例：
  - "512四单一组10" → 号码512，4注直选，1组，金额10元
  - "738 三十单 60" → 号码738，30注直选，金额60元
  - "777 十单 20" → 号码777，10注直选，金额20元

### 2.2 组三
- 格式：号码 + "组三" + 金额/注数
- 特征：3位数字中有2位相同（如：112、121、211、665等）
- 示例：
  - "899 三组 6" → 号码899，组三玩法，3组，6元
  - "23457组三20" → 组选号码23457，组三玩法，20元

### 2.3 组六
- 格式：号码/选号 + "组六" + 金额/注数
- 特征：3位数字各不相同，或从多个数字中选3个组合
- 示例：
  - "23457组六30" → 从23457中选3个数字的组六，30元
  - "02479组六50" → 从02479中选3个的组六，50元
  - "14578组六10" → 从14578中选3个的组六，10元

### 2.4 复式
- 关键词："复试"、"复式"
- 示例："479复试一单12" → 号码479，复式投注，1注，12元

### 2.5 定位/二码定位
- 格式：X*Y 或 X码定位
- 示例：
  - "3*8" → 定位投注，某位为3，某位为8
  - "福2码定位各10米" → 福彩2码定位，每个10元

### 2.6 多号码批量
- 多个号码用逗号、点号、空格分隔
- 示例：
  - "149.914.941三单一组24米" → 3个号码，每个3单1组，共24元
  - "409  908各四单一组20" → 2个号码，各4单1组，共20元
  - "660 661 662...669各五单100" → 10个号码，各5单，共100元

## 三、金额与单位识别

1. **金额表达**：
   - 直接数字："10"、"20"、"100"
   - 带单位："10米"、"10元"、"￥200"
   - 总计："共24米"、"总共30"、"合计50"、"总14米"

2. **投注单位**：
   - "X单"：X注直选
   - "X组"：X组（组三或组六）
   - "一组"：通常=2注（直选+组选组合）
   - "X单一组"：X注直选 + 对应组选
   - "X单X组"：X注直选 + X组组选

3. **单注金额计算规则**：
   - 直选单式：2元/注
   - 组三：2元/注（单选3个排列）
   - 组六：2元/注（单选1个组合）

## 四、解析输出格式

将每条报单解析为以下JSON格式：

```json
{
    "orders": [
        {
            "lottery_category": "FC",
            "game_type": "3D",
            "play_type": "直选|组三|组六|复式|定位",
            "numbers": ["512"],
            "bet_count": 4,
            "group_count": 1,
            "amount": 10.00
        }
    ],
    "total_amount": 10.00,
    "parse_status": "SUCCESS|PARTIAL|FAILED",
    "parse_msg": ""
}
```

## 五、字段说明

- **lottery_category**: "FC"(福彩) 或 "TC"(体彩)
- **game_type**: "3D"(福彩3D) 或 "P3"(排列三)
- **play_type**: 玩法类型
  - "直选"：直接选号
  - "组三"：组选三（有重复数字）
  - "组六"：组选六（无重复数字）
  - "复式"：复式投注
  - "定位"：定位投注
  - "组选"：未明确区分组三组六时使用
- **numbers**: 号码数组，每个号码为3位字符串
- **bet_count**: 投注注数（"X单"中的X）
- **group_count**: 组数（"X组"中的X）
- **amount**: 该条投注的金额（元）
- **total_amount**: 所有投注的总金额
- **parse_status**: 解析状态
  - "SUCCESS": 完全解析成功
  - "PARTIAL": 部分解析成功
  - "FAILED": 解析失败
- **parse_msg**: 解析备注或错误信息

## 六、特殊情况处理

1. **组选号码展开**：
   - "23457组六" → 从5个数字中选3个，共C(5,3)=10种组合
   - 返回时numbers只保留原始输入："23457"

2. **批量号码**：
   - 多个号码逐一解析，每个号码生成一条order记录
   - 或标注号码数量

3. **含总计行**：
   - "总14米"、"共24米" → 用于校验，放入total_amount

4. **无法识别的文本**：
   - 聊天内容、无关文字 → parse_status="FAILED"

## 七、重要提示

1. 仅输出JSON，不要有任何额外文字、解释或markdown格式
2. 金额保留2位小数
3. 号码统一为字符串格式
4. 无明确彩种标识时默认为"FC"/"3D"
5. 一条消息可能包含多个投注，全部解析到orders数组中
6. 遇到"总XX"、"共XX"、"合计XX"时提取为total_amount用于校验
```

### 6.5 API调用实现

#### 6.5.1 请求构造

```java
@Service
@Slf4j
public class AiParseServiceImpl implements AiParseService {

    @Value("${ai.base-url}")
    private String aiBaseUrl;

    @Value("${ai.api-key}")
    private String aiApiKey;

    @Value("${ai.model}")
    private String aiModel;

    @Value("${ai.timeout}")
    private int timeout;

    private static final String SYSTEM_PROMPT = """
        你是一个专业的中国彩票报单解析助手...
        (完整System Prompt内容)
        """;

    @Async("aiParseExecutor")
    public void parseAndSave(Long rawId, String rawText) {
        try {
            AiParseResult result = callAiWithRetry(rawText, 3);
            if (result.isSuccess()) {
                saveNormalizedOrder(rawId, rawText, result);
                updateNumberStats(result);
                statsNotifyService.notify(result.getLotteryCategory(), result.getIssueNo());
            } else {
                saveFailedOrder(rawId, rawText, result.getErrorMsg());
                saveToFailQueue(rawId, rawText, result.getErrorMsg());
            }
        } catch (Exception e) {
            log.error("AI解析异常, rawId={}", rawId, e);
            saveToFailQueue(rawId, rawText, e.getMessage());
        }
    }

    private AiParseResult callAiWithRetry(String rawText, int maxRetries) {
        Exception lastException = null;
        for (int i = 0; i < maxRetries; i++) {
            try {
                if (i > 0) {
                    long sleepMs = (long) Math.pow(2, i) * 1000; // 1s, 2s, 4s
                    Thread.sleep(sleepMs);
                    log.info("AI解析第{}次重试, 等待{}ms", i + 1, sleepMs);
                }
                return doCallAi(rawText);
            } catch (Exception e) {
                lastException = e;
                log.warn("AI解析第{}次失败: {}", i + 1, e.getMessage());
            }
        }
        return AiParseResult.fail("超过最大重试次数: " + lastException.getMessage());
    }

    private AiParseResult doCallAi(String rawText) {
        // 构造请求体
        JSONObject body = new JSONObject();
        body.put("model", aiModel);
        body.put("temperature", 0.1);
        body.put("max_tokens", 500);

        JSONArray messages = new JSONArray();
        messages.add(Map.of("role", "system", "content", SYSTEM_PROMPT));
        messages.add(Map.of("role", "user", "content", rawText));
        body.put("messages", messages);

        // 发送请求
        String responseStr = HttpUtil.createPost(aiBaseUrl)
            .header("Authorization", "Bearer " + aiApiKey)
            .header("Content-Type", "application/json")
            .body(body.toJSONString())
            .timeout(timeout)
            .execute()
            .body();

        log.debug("AI响应: {}", responseStr);

        // 解析响应
        JSONObject response = JSON.parseObject(responseStr);

        // 检查是否有错误
        if (response.containsKey("error")) {
            String errorMsg = response.getJSONObject("error").getString("message");
            throw new RuntimeException("AI API错误: " + errorMsg);
        }

        // 提取content
        String content = response.getJSONArray("choices")
            .getJSONObject(0)
            .getJSONObject("message")
            .getString("content");

        // 清理可能的markdown格式
        content = cleanJsonContent(content);

        // 解析JSON
        JSONObject parsed = JSON.parseObject(content);

        // 检查是否解析失败
        if (parsed.containsKey("error")) {
            return AiParseResult.fail(parsed.getString("reason"));
        }

        return AiParseResult.success(parsed);
    }

    private String cleanJsonContent(String content) {
        // 移除可能的markdown代码块标记
        content = content.trim();
        if (content.startsWith("```json")) {
            content = content.substring(7);
        } else if (content.startsWith("```")) {
            content = content.substring(3);
        }
        if (content.endsWith("```")) {
            content = content.substring(0, content.length() - 3);
        }
        return content.trim();
    }
}
```

### 6.6 解析结果校验

AI返回结果后，需进行业务校验：

```java
public class OrderValidator {

    public static ValidationResult validate(AiParseResult result) {
        List<String> errors = new ArrayList<>();

        // 1. 校验彩种
        String category = result.getLotteryCategory();
        String gameType = result.getGameType();
        if (!isValidGameType(category, gameType)) {
            errors.add("无效的彩种组合: " + category + "/" + gameType);
        }

        // 2. 校验号码格式
        List<String> numbers = result.getNumbers();
        String validationError = validateNumbers(gameType, numbers);
        if (validationError != null) {
            errors.add(validationError);
        }

        // 3. 校验金额
        if (result.getBetAmount().compareTo(BigDecimal.ZERO) <= 0) {
            errors.add("投注金额必须大于0");
        }
        if (result.getMultiple() < 1 || result.getMultiple() > 99) {
            errors.add("倍数必须在1-99之间");
        }

        return errors.isEmpty()
            ? ValidationResult.success()
            : ValidationResult.fail(String.join("; ", errors));
    }

    private static String validateNumbers(String gameType, List<String> numbers) {
        return switch (gameType) {
            case "SSQ" -> validateSSQ(numbers);  // 6红+1蓝
            case "DLT" -> validateDLT(numbers);  // 5前+2后
            case "3D", "P3" -> validateThreeDigits(numbers);
            case "P5" -> validateFiveDigits(numbers);
            case "QLC" -> validateQLC(numbers);  // 7个号码
            case "QXC" -> validateQXC(numbers);  // 7位数字
            default -> "未知彩种";
        };
    }

    private static String validateSSQ(List<String> numbers) {
        // 查找分隔符位置
        int separatorIndex = numbers.indexOf("|");
        if (separatorIndex != 6) {
            return "双色球红球数量应为6个";
        }
        if (numbers.size() != 8) { // 6红 + | + 1蓝
            return "双色球号码格式错误";
        }
        // 校验红球范围01-33
        for (int i = 0; i < 6; i++) {
            int num = Integer.parseInt(numbers.get(i));
            if (num < 1 || num > 33) {
                return "双色球红球范围应为01-33";
            }
        }
        // 校验蓝球范围01-16
        int blue = Integer.parseInt(numbers.get(7));
        if (blue < 1 || blue > 16) {
            return "双色球蓝球范围应为01-16";
        }
        return null;
    }

    // ... 其他彩种校验方法
}

### 6.7 重试与失败处理

#### 6.7.1 重试策略

| 参数 | 值 | 说明 |
|------|-----|------|
| 最大重试次数 | 3次 | 首次调用 + 2次重试 |
| 退避策略 | 指数退避 | 1s → 2s → 4s |
| 单次超时 | 30秒 | HTTP请求超时 |
| 可重试错误 | 网络超时、5xx错误、限流 | 4xx错误不重试 |

#### 6.7.2 失败队列处理

```
3次实时重试失败后：
  ├── 写入 t_order_normalized (ai_parse_status=2)
  └── 写入 t_ai_parse_fail_queue (status=0)

定时任务 AiParseRetryTask (每60秒)：
  ├── 查询 status=0 且 retry_count<10 的记录
  ├── 逐条重新调用AI解析
  ├── 成功 → status=1, 更新t_order_normalized
  ├── 失败 → retry_count++
  └── retry_count>=10 → status=2 (放弃，需人工处理)

管理接口：
  ├── GET /api/admin/parse-queue → 查看失败队列
  ├── POST /api/admin/parse-queue/{id}/retry → 手动重试单条
  └── POST /api/admin/parse-queue/retry-all → 批量重试
```

### 6.8 性能优化建议

1. **Prompt缓存**：System Prompt较长，可考虑使用支持Prompt缓存的模型以降低token消耗
2. **批量解析**：如报单量大，可考虑批量发送（需评估模型支持情况）
3. **本地预校验**：在调用AI前，先用正则做简单预筛选，明显不是报单的文本直接跳过
4. **结果缓存**：相同报单文本可缓存解析结果（需考虑期号变化）

### A.2 DTO类

#### AiParseResult.java
```java
@Data
@Builder
public class AiParseResult {
    private List<OrderItem> orders;
    private BigDecimal totalAmount;
    private String parseStatus;  // SUCCESS/PARTIAL/FAILED
    private String parseMsg;

    @Data
    @Builder
    public static class OrderItem {
        private String lotteryCategory;  // FC/TC
        private String gameType;         // 3D/P3
        private String playType;         // 直选/组三/组六/复式/定位
        private List<String> numbers;    // 号码列表
        private Integer betCount;        // 注数（X单）
        private Integer groupCount;      // 组数（X组）
        private BigDecimal amount;       // 金额
    }

    public boolean isSuccess() {
        return "SUCCESS".equals(parseStatus) || "PARTIAL".equals(parseStatus);
    }

    public static AiParseResult fromJson(JSONObject json) {
        AiParseResult result = new AiParseResult();
        result.setParseStatus(json.getString("parse_status"));
        result.setParseMsg(json.getString("parse_msg"));
        result.setTotalAmount(json.getBigDecimal("total_amount"));

        JSONArray ordersJson = json.getJSONArray("orders");
        List<OrderItem> orders = new ArrayList<>();
        if (ordersJson != null) {
            for (int i = 0; i < ordersJson.size(); i++) {
                JSONObject o = ordersJson.getJSONObject(i);
                OrderItem item = OrderItem.builder()
                    .lotteryCategory(o.getString("lottery_category"))
                    .gameType(o.getString("game_type"))
                    .playType(o.getString("play_type"))
                    .numbers(o.getJSONArray("numbers").toJavaList(String.class))
                    .betCount(o.getIntValue("bet_count", 1))
                    .groupCount(o.getIntValue("group_count", 0))
                    .amount(o.getBigDecimal("amount"))
                    .build();
                orders.add(item);
            }
        }
        result.setOrders(orders);
        return result;
    }

    public static AiParseResult fail(String msg) {
        return AiParseResult.builder()
            .orders(new ArrayList<>())
            .totalAmount(BigDecimal.ZERO)
            .parseStatus("FAILED")
            .parseMsg(msg)
            .build();
    }
}
```
